# Ovo 产品哲学

> 这份文档是 Ovo 的产品宪法。它先于代码，先于功能，先于路线图。
> 当任何一次产品决策出现分歧时，回到这里。

---

## 一、一句话定义

> **Ovo 是市面上第一个敢主动替你做事、又全程让你看着、随时能教它的桌面 AI。**
>
> ——玻璃房子里的主动管家。

我们不是另一个 ChatGPT 桌面，不是另一个 Rewind，不是另一个 Copilot。

我们是**唯一一个把"主动"和"透明"捏在一起做的 AI 助手**。

---

## 二、产品信念（不可妥协）

### 1. 主动是灵魂
被动等待用户提问的 AI 已经被 ChatGPT/Claude 桌面占满。Ovo 存在的理由只有一个：**在你需要它之前就到位**。

用户已经验证过 Ovo 能做到——它能预测下一步行为、知道该回复别人什么内容、理解当前的工作场景。这是真正稀缺的产品资产。**任何让 Ovo 变得更被动的决策都是退步。**

### 2. 透明是骨架
主动 AI 的死法只有一种：**用户失去控制感**。Recall 死于此，Personal AI 死于此。

我们的解法不是降低自动化，而是让**透明度跑在自动化前面**——用户先看见 AI 怎么想，才会授权它怎么做。

透明不是一个 feature，不是侧边栏的调试视图——**透明是产品本身**。

### 3. 教练是闭环
记录用户反馈不等于学习。如果用户无法直接告诉 AI "这种时刻多来一些"、"那种时刻永远不要做"，所有的"学习能力"都是幻觉。

**用户必须能直接、可见地塑造 Ovo 的行为。**

---

## 三、核心框架：魔法时刻工程化

Ovo 现在的状态是"偶发的魔法"。我们要做的工程，是把它变成"可复现的魔法"。

> **魔法 = 命中率 × 命中价值 × 用户感知到命中**

所有产品工作都围绕这三根轴。任何不服务于这三轴的 feature，都该被砍掉。

### 轴 1：命中率工程（什么时候出手 vs 沉默）

**当前问题**：pipeline 几乎是"看到 OCR 变化就跑"。大部分时刻用户**不需要 AI 出手**，每次"无效出手"都在消耗"有效出手"的可信度。

**打磨方向**：
- **意图密度阈值**：基于 KG 的 entity activation 强度，只有"用户正在做一件可被 AI 帮上忙的事"时才推理
- **沉默是金作为目标函数**：feedback-engine 不只学"被接受的建议"，也要学"用户在场但没接受 = 这种时刻不应该出手"
- **主动度自调节**：连续 N 次忽略 → 自动降级；连续 N 次接受 → 自动升级

### 轴 2：命中价值工程（出手时给的东西够好）

**当前优势**：多 pass prompt + KG insight + 人格画像融合，已经能产生 wow moment。这是 Ovo 的天赋，必须守住。

**打磨方向**：
- **人格维度扩展**：从当前 2 维（技术专注 + 多任务）扩展到沟通语气 / 决策模式 / 对不同人的关系亲疏度（KG 已有底子）
- **中间推理可修正**：多 pass 的中间产物（intent / entity）不能直接喂下一 pass，要在 Reflect 层让用户能修正
- **守住"理解我"的天花板**：每一个降低上下文丰富度的优化都要慎重——速度可以妥协，理解力不能

### 轴 3：命中感知工程（用户能感知到"它救了我"）

**最被低估的一根轴**。魔法不被看见 = 没有发生。

**打磨方向**：
- **即时价值反馈**：建议被采纳后，浮窗显示"这次 Ovo 帮你省了 ~3 分钟"
- **周报式量化**：本周 Ovo 出手 X 次，命中 Y 次，省了 Z 分钟——把魔法量化、可炫耀、可记忆
- **魔法时刻存档**：用户标记的"这次太准了"自动收藏，成为 prompt-self-eval 的正样本

## 四、产品架构：三层嵌套

```
        ┌─────────────────────────────┐
        │   ⚡ 主动层（Act）           │
        │   ┌───────────────────┐     │
        │   │ 🪟 透明层（Trace） │     │
        │   │   ┌─────────────┐ │     │
        │   │   │ 🎓 教练层    │ │     │
        │   │   │  (Reflect)  │ │     │
        │   │   └─────────────┘ │     │
        │   └───────────────────┘     │
        └─────────────────────────────┘
```

**每一次主动行为必经三态**：

| 阶段 | 含义 | 用户体验 |
|---|---|---|
| **Act** | AI 出手（主动观察、推理、建议、执行） | "它在帮我做事" |
| **Trace** | 出手全程留痕，可回溯（OCR → KG → prompt → LLM → action） | "我看得见它怎么做的" |
| **Reflect** | 用户可以修正、教育、设置规则 | "我能让它越来越懂我" |

**当前 codebase 状态**：Act 和 Trace 的骨架已有，Reflect 几乎为零——这是接下来最大的杠杆。

## 五、关键产品机制

### 机制 1：信任分级（不是"自动执行白名单"）

废除"3 种 auto-execute、6 种 pending"的写死逻辑。改成用户可视化、可拖动的信任阶梯：

```
🔓 Lv.0 仅展示（建议出现在面板，不动手）
🔓 Lv.1 草拟（替我准备好草稿，等我点发）
🔓 Lv.2 一键确认（弹窗"要发这封吗？"大按钮）
🔓 Lv.3 默认执行 + 5 秒撤销（Gmail 撤销发送模式）
🔓 Lv.4 完全托管（仅特定 action + 特定时段）
```

每个 action 类型可独立调档。**主动度是用户授权出来的，不是产品默认设的。**

### 机制 2：玻璃管家浮窗

AI 主动出手时浮窗显示：

```
⚡ Ovo 正在主动操作
  ↳ 看见了：Chrome - Gmail 草稿（OCR 摘要）
  ↳ 想做：复制收件人到剪贴板
  ↳ 因为：你过去 3 次在此场景都这么做
  [让它做] [不要做] [永远不要在这种情况做]
```

"永远不要…"按钮 = 用户主动训练 AI。点击后 prompt-self-eval 触发，生成新约束规则，写入 KG 作为 negative pattern。

### 机制 3：驯化曲线替代 Bootstrap 问卷

第一次进入不再问"你关心什么"。改成一句话承诺：

> **"Ovo 默认只看不做。当我观察到某个模式重复 3 次，会问你：要不要我下次替你做？"**

把 onboarding 从"灌输偏好"变成"承诺克制"。

### 机制 4：Pipeline 详情升为主时间线

当前 PipelineDetail 藏在二级页面，是"调试视图"。升级为"AI 行为流"主时间线——每条主动行为可点开看完整因果链。

**这是 Ovo 区别于 Rewind/Recall 的最强卖点之一**，不能藏。

## 六、入场承诺（第一句话）

用户打开 Ovo 看到的第一句话定义了全部预期。我们选：

> **"看着 Ovo 思考。它每一步都会告诉你为什么。"**

这句话同时承诺了**主动**（它在思考 = 它在工作）和**透明**（每一步告诉你为什么）。

**这是承诺，做不到就是骗子。**

---

## 七、决策判准（每个 PR 必问）

当你不确定一个 feature / 改动该不该做时，回答这三个问题：

1. **它服务于哪一轴？** 命中率 / 命中价值 / 命中感知 — 如果都不沾，砍。
2. **它在 Act / Trace / Reflect 三层中处于哪层？** 如果只增强了 Act 而没有同步的 Trace，警惕——这会加剧"用户失去控制感"。
3. **它兑现了"看着 Ovo 思考"的承诺吗？** 如果它让 Ovo 变得更不透明，重新设计。

---

## 八、明确不做的事（反向定义）

- ❌ **不做"AI 帮你刷消息"的全自动化 agent**——那是另一个赛道，会冲淡核心定位
- ❌ **不做"AI 个人秘书"的拟人化包装**——拟人化只会拉高无法兑现的预期
- ❌ **不做"魔法体验"的黑盒**——任何"猜不到为什么"的 AI 行为都违反产品哲学
- ❌ **不为速度牺牲理解力**——多 pass 慢一点没关系，命中价值不能塌
- ❌ **不做"先收集数据，未来再用"的承诺**——隐私是产品基础，不是营销文案

## 九、竞争对照

| 产品 | 死法 | Ovo 的解 |
|---|---|---|
| Rewind | 能回放但帮不了你 | Act 层主动 + Reflect 层可教 |
| Microsoft Recall | 能看但你不知道我看了什么 | Trace 层全程透明 |
| Personal AI | 承诺懂你但永远懂不了 | 用户可直接修正中间推理 + KG 主动可视化 |
| ChatGPT 桌面 | 必须用户先问 | 主动出手 + 命中感知量化 |
| Cursor 等 IDE Copilot | 只在 IDE 内 | 跨所有窗口的全局上下文融合 |

---

## 十、写给未来的自己

如果某天 Ovo 变得不再透明、不再可教，只剩"主动建议"——那已经不是 Ovo 了，是又一个被市场淘汰的 AI 副驾驶。

如果某天 Ovo 变得完全透明、完全可教，但失去了主动出手的勇气——那也不是 Ovo 了，是一个加了花哨日志的 ChatGPT。

**主动 + 透明 + 教练，三者缺一不可。这是 Ovo 存在的理由。**
