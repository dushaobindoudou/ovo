# ovo 接续开发计划（可直接执行）

更新时间：2026-04-15

本计划用于下次直接续开发，不依赖聊天上下文。

## P0（优先级最高）

### 1. 后台窗口并行捕获增强

目标：不仅捕获活动窗口，还按用户选择并行捕获后台窗口内容。

建议步骤：

1. 在 `AutoCaptureService` 中引入“活动窗口 + monitoredWindows”双通道采集。
2. 每个窗口独立维护 `WindowBuffer`，保持 `windowId::appName` 聚合策略。
3. Pipeline 中新增“窗口来源分布”统计字段。
4. UI 窗口面板增加每个监听窗口的最近捕获时间和失败率。

验收标准：

- 至少 2 个窗口并行采集稳定运行 10 分钟无崩溃。
- Pipeline 中可看到多窗口增量数据。

### 2. Action 执行确认流完善

目标：把 pending Action 的确认-执行-回写链路做完整。

建议步骤：

1. 在建议面板增加 pending Action 列表区域。
2. 新增确认弹窗（执行/取消）。
3. 执行后将结果回写 `pipeline_logs.stages.actions`。
4. 失败时给出可重试按钮与错误详情。

验收标准：

- pending Action 可以在前端完成确认并触发执行。
- 执行结果可在 Pipeline 详情看到。

### 3. Agent 结果结构化校验

目标：降低 LLM 输出不规范导致的失败。

建议步骤：

1. 对 `response.raw` 增加 JSON schema 校验。
2. 校验失败时引入“二次修复提示词”重试一次。
3. 无法修复时按降级策略只保留 `content` 文本建议。

验收标准：

- 非 JSON 返回不再导致全链路中断。
- Pipeline 明确记录“schema 修复”阶段。

## P1（第二优先级）

### 4. 知识图谱质量优化

1. 实体去重策略（别名、大小写、同义词）。
2. 关系强度衰减与时效策略（长期无证据则降权）。
3. 新增图谱查询 API：按实体查看近期证据事件。

### 5. 人格画像可解释性增强

1. 人格每个维度展示证据来源（来自哪些事件和应用）。
2. 增加手动修正入口（用户可直接调整偏好）。
3. Prompt 注入用户手动偏好优先于自动分析。

### 6. 运行稳定性

1. 将定时器（capture/agent/health）统一纳入调度器管理。
2. 加入内存与队列长度监控，防止长时运行堆积。
3. 增加关键模块的异常恢复策略与告警等级。

## P2（第三优先级）

### 7. 端到端自动化测试

1. 新增最小 e2e：启动 -> 真实捕获（需权限） -> 触发 pipeline -> 断言 UI 渲染。
2. 增加 agent smoke 的“严格模式”（SKIP 视为失败）。
3. 在 CI 中输出测试报告与构建产物摘要。

### 8. 打包与发布流程

1. 增加发布脚本（版本号、changelog、打包命令）。
2. 增加发布前检查清单。
3. 规划 auto-update（后续可选）。

## 下次开发快速启动

```bash
pnpm install
pnpm approve-builds
pnpm test:ci
pnpm dev
```

建议先做 `P0-1` 与 `P0-2`，这两项对可用性提升最大。
