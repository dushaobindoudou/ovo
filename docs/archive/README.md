# 文档归档说明

更新时间：2026-05-24

这里保存历史审计、旧设计稿、旧状态快照和传播素材。归档文档不代表当前实现状态，也不作为当前 backlog 的唯一依据。

## 目录

- `audits/`：历史 UX / UI / Bug / Review 报告。很多问题已经修复，当前待办以 `../UNRESOLVED_ISSUES.md` 和 `../product/PRODUCT_EXPERIENCE_BACKLOG.md` 为准。
- `specs/`：2026-04 的 nudge、OCR、Claude 集成设计稿，保留用于追溯早期方案。
- `history/`：旧实现状态和复盘记录。
- `community/`：Founder story 等社区传播资料。

## 使用原则

1. 需要理解历史决策时读归档。
2. 需要排当前优先级时读当前文档。
3. 不要把归档里的 P0/P1 直接搬进开发计划，必须先对照当前代码验证是否仍存在。
